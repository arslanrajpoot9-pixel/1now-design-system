Mobile table — 390px horizontal scroll v1.5.1
Mobile table rule
On 390px screens, data tables never wrap — they scroll horizontally inside an overflow-x:auto container.
The first column is sticky (position:sticky;left:0) so users always know which row they're reading.
A right-edge gradient signals there is more content to scroll to.
Tables never card-convert on mobile — data integrity is more important than compact layout. Always scroll, never wrap.

Rules
Don'ts — 34 rules
33 non-negotiable rules
All 33 rules
✘
Invent new colors. Palette is closed.
✘
Fill the sidebar or large surfaces with brand orange. Orange is an accent only.
✘
Use gradients anywhere except the approved area-chart fill.
✘
Add drop shadows to default cards. Border only.
✘
Use color alone to convey state — always pair with text or icon.
✘
Use Tailwind gray-* or any cool neutrals. Warm greys only.
✘
Use emoji in product UI.
✘
Use arbitrary spacing (7px, 22px, 18px). 4px grid only.
✘
Add scale hover pops, spring bounces, or glow effects.
✘
Mix 1.5px and 2px icon strokes in the same view.
✘
Put a chart legend inside the SVG. Always external HTML.
✘
Use rose, pink, red, blue, or green as table row backgrounds.
✘
Use #3D3D3A in single-platform charts. Reserved for cross-platform comparison only.
✘
Use colored row backgrounds to group categories. Use 2px left-border accent instead.
✘
Use semantic colors for feature/module tags on kanban cards.
✘
Render nav items, table cells, or buttons below font-weight 500.
✘
Apply the hero card edge (3px top border) to any secondary element.
✘
Show Atlas data to client-facing users. Role access is mandatory.
✘
Use more than 4 color stops in any donut. Group extras as "Other".
✘
Use ALL CAPS anywhere except 11px overline labels. Sentence case everywhere else.
✘
Show profile/avatar in more than one place. The user avatar and name appear in the topbar only — top-right corner, on every surface. Never show the avatar, name, or email in the sidebar, inner panels, or Carisma's chat sidebar. Duplicating profile identity breaks layout consistency and confuses which instance is authoritative.
✘
Customize the sidebar/topbar shell per surface. All three surfaces (app.1now.ai, Carisma, Atlas) share the identical shell — same sidebar structure, same topbar, same spacing, same token usage. Only the content area changes. Never add surface-specific chrome to the shell itself. If something must differ, it lives inside the content area, not the shell.
✘
Use PNG/JPG for the logo. The 1Now logo must be inline SVG in all HTML surfaces. SVG scales perfectly at any resolution, loads without network requests, and never blurs. Never link to an external image file for the logo.
✘
Recolor the scooter icon. The vehicle icon mark is always #FE7743 orange — no exceptions. Never render it in grey, black, white, or any other color. The bolt icon (sidebar shorthand) follows the same rule: always orange on white background.
✘
Use arbitrary spacing values. All margins, padding, and gaps must come from the 4px base grid: 4 · 8 · 12 · 16 · 20 · 24 · 28 · 32 · 40 · 48 · 64px. Never use 5px, 7px, 10px (except 10px border-radius), 15px, 18px, or any value not on the 4px grid.
✘
Mix card metric sizes in the same row. All KPI cards in a row must use the same value size. The standard is 24px/600. Hero card gets 36px/700. Alert cards get 28px/700. Never put a 32px value card next to a 24px value card in the same row — it breaks the visual rhythm.
✘
Use bold for body copy. Font weight 700 is for hero values only. 600 is for card values, overlines, active nav, and deltas. 500 is for nav items and UI labels. 400 is for body text, subtitles, and descriptions. Never bold a paragraph or description block.
✘
Use cool greys anywhere in the UI. The entire 1Now palette is warm — #ECECE8, #FAFAF8, #F7F7F5, #E7E7E2. Never introduce a neutral grey with blue or green undertones (like #DADED6 or #E0E0E0). If a new surface color is needed, derive it from the warm palette by adjusting lightness only.
✘
Use semantic colors for decorative purposes. Red (#B52B2B) = error/danger only. Green (#2D7A4F) = success only. Yellow (#8C5A00) = warning only. Blue (#1A5DA8) = info only. Never use these colors as accent, decoration, or branding. Orange (#FE7743) is the only brand accent.
✘
Mix border-radius scales in the same component family. Cards use 12–16px. Inputs use 8px. Badges use 99px pill. Buttons use 8–10px. Nav items use 8px. Never put a 4px radius card next to a 16px radius card. Radius must be consistent within a component type.
✘
Mix icon libraries or use filled icons. Every icon in 1Now is from Tabler Icons — outline only. Never use filled, duo-tone, or solid icon variants. Never mix Tabler with Heroicons, Lucide, Feather, or any other library. One library, one style, everywhere.
✘
Add shadow to elements that are not floating. Shadow-float is reserved for the sidebar only — it signals elevation above the page. Cards use a very subtle 1px border + optional 0 1px 4px shadow. Never give a shadow to a table row, badge, tab, or inline element. Shadow means "this floats above the canvas."
✘
Use flex or grid layout in email templates. HTML emails must use table-based layout only for Outlook/Gmail/Apple Mail compatibility. Never use CSS flexbox, grid, or position:absolute inside email HTML. Every layout must degrade gracefully to a simple single-column stack on old clients.
✘
Use Tabler icon webfont in emails. Webfonts are blocked by most email clients. All email icons must be inline SVG paths. Never reference cdn.jsdelivr.net or any external font/icon URL inside an email template.
